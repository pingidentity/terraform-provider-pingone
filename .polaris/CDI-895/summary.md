# Summary

## What Was Built
Fix `pingone_application` CORS origins validator to match PingOne API limit of 40

The `oidc_options.cors_settings.origins` attribute previously enforced a Terraform-side maximum of 20 entries, but the PingOne API allows up to 40. This raised the `originsMax` constant from 20 to 40 in `resource_application.go` (propagating automatically to the validator and description string), updated the independent hardcoded description string in `data_source_application.go`, and regenerated provider docs via `make generate`.

## Key Decisions
- **Single constant as authoritative limit**: `originsMax` in `resource_application.go` is the sole source of truth for the resource schema. Changing it updates both the `setvalidator.SizeAtMost` call and the `fmt.Sprintf` description string automatically.
- **Data source requires an independent edit**: `data_source_application.go` does not reference `originsMax` — it uses a standalone string literal. A separate edit was required.
- **Schema upgrade file intentionally untouched**: `resource_application_schema_upgrade_0_to_1.go` represents the V0 schema shape for state migration and correctly has no validator on `origins`.
- **Docs regenerated, not manually edited**: `docs/resources/application.md` and `docs/data-sources/application.md` are auto-generated by `tfplugindocs` via `make generate`. Both now reflect "Limited to 40 values.".

## Changes
- `internal/service/sso/resource_application.go` — changed `const originsMax = 20` to `const originsMax = 40`; validator and description string auto-updated
- `internal/service/sso/data_source_application.go` — updated hardcoded "Limited to 20 values." to "Limited to 40 values." at line 776
- `docs/resources/application.md` — regenerated; 3 occurrences of "Limited to 40 values." (was 20)
- `docs/data-sources/application.md` — regenerated; 3 occurrences of "Limited to 40 values." (was 20)

## Testing
`go build ./...` and `go vet ./...` pass cleanly. No existing test asserts the old 20-origin limit so no regressions are possible at the unit level. Acceptance tests require live PingOne credentials and were not run; the change is a pure validator/string update with no API behavior change. All 7 acceptance criteria verified by QA. Holistic review: APPROVE.
